App Name: BookMate

2. Purpose

A program that allows customers to book a time slot on a calendar for a window cleaning or landscaping service. It notifies the business owner of the job details, including price, location, and date/time.

3. Data Description

Appointments Table

Purpose: Stores appointment details to manage service bookings.

Information Stored: Includes date, time, type of service, customer, and job status.

  • appointment_id (INT, PRIMARY KEY, AUTO_INCREMENT) - Unique identifier
  • date (DATE) - Scheduled appointment date
  • time (TIME) - Scheduled appointment time
  • service_type (VARCHAR(100)) - Type of service requested
  • customer_id (INT, FOREIGN KEY) - Links to the customer
  • status (ENUM('pending', 'confirmed', 'completed')) - Appointment status

Customers Table

Purpose: Stores customer information for appointment tracking.

Information Stored: Includes customer details such as name, phone, and email.

  • customer_id (INT, PRIMARY KEY, AUTO_INCREMENT) - Unique identifier
  • name (VARCHAR(255)) - Full name of the customer
  • phone_number (VARCHAR(20)) - Customer phone number
  • email (VARCHAR(255)) - Customer email address

4. Functions Performed by the App

  • Book a date and time - Allows users to select an available slot for a service.
    Database effect: INSERTS a new appointment into the appointments table.
  • Modify a booking - Users can update their booking details (e.g., change date/time).
    Database effect: UPDATES an existing record in the appointments table.
  • Cancel a booking - Users can cancel a scheduled appointment.
    Database effect: DELETES the record from the appointments table.
  • View available time slots - Displays open booking slots for services.
    Database effect: SELECTS available time slots from the appointments table.
  • View customer details - Admins can see customer contact info for bookings.
    Database effect: SELECTS customer details from the customers table.
  • Send confirmation notifications - Notifies users and business owners about confirmed bookings.
    Database effect: No direct effect on the database (handled via external email/SMS service).

5. Role of PHP and JavaScript

PHP: Handles backend operations, such as storing and retrieving booking data from the database, processing form submissions, and sending email notifications.

JavaScript: Provides interactivity, such as dynamic calendar updates, form validation, and real-time availability checks without reloading the page.
